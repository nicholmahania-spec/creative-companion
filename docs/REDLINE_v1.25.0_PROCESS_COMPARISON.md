# Redline v1.25.0 — Pro design process vs product (crayon-level mapping)

Compared against a professional 7-step graphic design process (Define → Research → Ideate → Sketch → Design → Review → Deliver).

## Gap table

| Pro step | What a designer does | Product today | Gap → fix |
|----------|----------------------|---------------|-----------|
| **1 Define** | Questions: who, feel, size, deadline; **one-sentence goal**; must/nice; Design Detective Sheet | Name + free brief + kits | **Detective sheet fields** (goal, audience, feel, must/nice, format) |
| **2 Research** | Client past, competitors, audience cues; mood board; **20-min timer** | Mood board + stars; timer off-path | **Research spy checklist + 20-min timer CTA** on Research |
| **3 Ideate** | 5–8 messy directions; opposites; no first-love | Sparks + A/B/C | **Spark progress toward 8** + opposite prompt; keep A/B/C shortlist |
| **4 Sketch** | 2–3 cleaner drafts + why; **&lt;2h** | One step + queue A/B/C drafts | **Process tip + why-on-draft** already partially there; reinforce |
| **5 Design** | Polish, hierarchy, **v1/v2 versions** | Artboard + lockups | **Design version field** (v1→v2) |
| **6 Review** | Specific questions; present; revise | Checklist + Helper + brief copy | **Feedback notes capture** + question chips |
| **7 Deliver** | Files, handover note, **eval paragraph** | Brand book PDF | **Handoff note + learnings** → PDF |

## Product honesty

Still a direction desk, not Figma. Goal = structure that matches real process so ADHD creatives don’t skip “measure the flour.”
