## Principles (apply bar)

1. **One next human move** in toast when something fails  
2. **Process words** = Define…Deliver only  
3. **XP** only inside Progress HUD / when prefs.on — never lead success toasts  
4. **Failures** = soft, fixable (“couldn’t…”, “try again”) not “failed” alone  
5. Keep length ADHD-short
